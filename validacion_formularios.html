//Validación de formularios
//Como ya hemos comentado la seguridad en el envío formularios es crucial para enviar ataques.
//Con HTML5 podemos hacer validaciones básicas en el lado del cliente (navegador) gracias a nuevos atributos:


//require: hace que un campo sea obligatorio.
//pattern: define una expresión regular para validar el formato de un campo.
//number: valida que sean números.
//min y max: especifica el valor mínimo y máximo para campos numéricos.
//email: valida que el campo sea un correo electrónico válido.
//maxlength: limita la cantidad máxima de caracteres.
//minlength: liminta la cantidad mínima de caracteres.
//tel: para campos telefónicos.
//url: para campos que tengan que ser una url.
//date: para campos que tengan fechas.
//NOTA.- Recuerda que la validación en el lado del cliente no es suficiente para garantizar la seguridad de tus datos, siempre se deben realizar validaciones adicionales en el lado del servidor.
